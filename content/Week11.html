Week11

課程內容 :

分組期末專案範例說明

說明如何利用 Heroku 雲端服務，伺服各分組的期末專題倉儲中的動態程式，並與 Github 倉儲內容保持自動

學習要點 :

雲端輔助設計網站

  • 規劃使用可攜 Heroku cli
  • 登記 Heroku 帳號, 完成信箱驗證後登入
  • 建立雲端網站倉儲
  • 將網站倉儲 git clone 到近端工作目錄
  • 配置近端工作目錄中的 Flask 輔助設計程式
  • 在近端登入 Heroku
  • 以 Git 新增提交推送, 將近端程式送至 Heroku 雲端後啟動
  • 將近端程式以 git remote add, 推送同步版本至 Github, Gitlab, Bitbucket 與區網中的 Fossil SCM
  • https://devcenter.heroku.com/articles/github-integration

Onshape 中的零組件輸入 V-rep 進行動態模擬

  • 將組立完成的零組件, 以 STL 格式轉出

  • 將 STL 組件輸入 V-rep 後, 將零組拆解

  • 根據零件運動軸進行設定

  • 以 Lua 嵌入或 Python remote API 方式驅動各致動器進行模擬

要讓程式能夠在 Heroku 雲端執行，必須要有這三個檔

  • runtime.txt - 指定python版本

  • Procfile - 如何啟動程式

  • requirements.txt

https://github.com/mdecourse/cadp2018/issues/17

心得 :

本週上課老師介紹了一些期末專案範例，像是3D列印機、行走機構與鋼球運動系統，並告知如要用其他軟體繪製只需將檔案轉為 stl 檔即可在 V-rep 開啟，也說明如何利用 Heroku 雲端服務伺服各分組的期末專題倉儲中的動態程式，並與 Github 倉儲內容保持自動。
